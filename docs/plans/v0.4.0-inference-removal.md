# v0.4.0 Plan — Strip the deprecated inference subsystem

> **Status: NOT EXECUTED. Plan only.**
>
> Drafted 2026-04-29 alongside the v0.3.8 deprecation notice. Re-verify
> against current `main` before executing — code shifts; this plan can rot.
> Especially: re-grep for `Giulia.Inference\.` and `Giulia.Provider\.`
> outside the deprecation footprint, since new code may add references.

## Goal

Remove the early-v0.x local-chat inference subsystem from Giulia. Cut as
**v0.4.0** — formally a breaking change (HTTP endpoints + MCP tools
removed). After the strip, Giulia is structurally read-only: no LLM
calls from inside the daemon, no write-tools, no approval flow, no
chat endpoint. The only LLM integration path is external clients
(Claude Code, Claude Desktop, anything speaking REST or MCP) calling
Giulia for data.

Estimated effort: **~2-3 hours focused work** + 30 min verification.

## Pre-flight checks (clear all before executing)

1. **No external dependency on the chat endpoint.** Confirm — including
   any of your own scripts, CI jobs, or tooling — that nothing calls
   `POST /api/command`, `POST /api/command/stream`, or any
   `/api/approval/*` / `/api/transaction/*` path. The user has stated
   internal use is gone; double-check before pulling.

2. **`Inference.Approval` interactive consent pattern.** Decide whether
   the per-action approval UX is worth preserving for any non-LLM use
   case (e.g., a future "preview before commit" flow on data writes).
   If yes, factor `Approval` out into a generic gate before deleting
   the inference layer. If no, it goes with the rest. Today's read:
   no, it goes.

3. **Re-run the dependency grep.** Verify no NEW callers have crept in
   since v0.3.8 deprecation:
   ```
   grep -rn "Giulia\.Inference\." lib/giulia --include="*.ex" \
     | grep -vE "lib/giulia/(inference|provider)/|lib/giulia/daemon/(endpoint\.ex|routers/(approval|transaction)\.ex)|lib/giulia/mcp/dispatch/(approval|transaction)\.ex|lib/giulia/tools/(patch_function|bulk_replace|rename_mfa)\.ex|lib/giulia/application\.ex"
   ```

   in `tools/trace_path.ex:38`, the word "inference" in

4. **Working tree clean** + on a fresh branch (`v0.4.0-strip` or similar).
   Do NOT do this on `main` directly; the diff is large and a

## Rollback

If something breaks during execution:

1. Working tree only: `git restore .` from the branch HEAD.
2. After a bad commit on the strip branch: `git reset --hard HEAD~1`.
3. After merge to main with a problem found post-deploy: `git revert
   <merge-commit>`. The change is a delete; revert is a re-add of
   the deleted code. Should be clean.
4. Production data (CubDB / ArcadeDB / ETS) is untouched by this
   change — no schema migration, no re-scan needed. Rollback restores
   the deleted code; data is portable across both versions.

## After the strip

   `@build N` → `@build N+1`.
